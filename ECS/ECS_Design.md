# 開発ルール: ECS (Entity Component System) アーキテクチャ

このプロジェクトはデータ指向設計(DOP)を採用しています。コードを書く際は以下のルールを厳守してください。

1. **データとロジックの分離**:
   - データは純粋な `struct` で定義すること。
   - ロジックは `ComponentSystem` を継承したクラスで実装すること。

2. **Chunkとアーキタイプ**:
   - エンティティのデータは必ず `Chunk` クラス内で管理すること。
   - `Position`, `Velocity` などのコンポーネントは独立した配列として定義し、SoA (Structure of Arrays) の構成をとること。

3. **ランダムアクセスの回避**:
   - システム内の処理は、必ず `Chunk` 内の配列を `for` ループで連続してアクセスすること。
   - エンティティIDによるランダム検索を避け、配列のインデックスを利用すること。

4. **重複禁止**:
   - データの整合性を保つため、同じエンティティのデータは必ず特定のChunkにのみ存在させること。
